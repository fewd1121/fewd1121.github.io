Responsive Web Design - Review

RWD:

Designing the optimal viewing experience no matter the device on which the site is being viewed

Static vs. Fluid Layouts:

Using %-based widths, margins, etc. instead of px-based

Media Queries:

Allow CSS rules targeted at specific variables, like screen-size

• When width > 1000px color text red

• When width < 1000px color text blue

Browser-Width Media Queries

Within CSS Stylesheet:


	@media screen and (min-width: 1024px) {
		color: red;
	}

Media Type: screen

Only target devices with a viewable screen

Rule: (mid-width: 1024px)

Only target windows larger than, or equal to, 1024px

Media Queries - Additional Examples

Don't have to specify a media type:


	@media (min-width: 700px) { ... }

Multiple rules:


	@media (min-width: 320px) and (max-width: 480px) { ... }

• These styles will only apply on devices between 320 and 480 pixels

Em Typography

EM ... What?

• An em is a unit of measurement

• Derived from print design

• Based on the size of the letter "M"

• Like pixels, they determine the size of elements on a web page

• Unlike pixels, which are fixed, ems are relative to their parent's font-size

• More Info

EM's

• Repeat: em's are relative to their parent's font-size

• 1em === the inherited font size

• If a <div> has font-size of 16px:

1em within said <div> is 16px

2em within said <div> is 32px

• If a <p> has font-size of 20px:

1em within said <p> is 20px

2em within said <p> is 40px

EM's

Example:


	body {
		font-size: 20px;
	}

	div {
		height: 5em; // 100px
		width: 3em; // 60px
		margin: 0.5em; // 10px
		border: 0.05em; // 1px
	}

EM's - General Guide Lines

1. Set a font-size on the <body> using pixels:


	body {
		font-size: 16px;
	}

2. Base other typography on ems:


	h1 {
		font-size: 3em; // 48px
	}

	h2 {
		font-size: 2em; // 32px
	}

3. Set font-size in media queries to be smaller:


	h1 {
		font-size: 1.5em; // 32px;
	}

<meta> Tag

<meta> Tag

• Mobile browsers render webpages in a window wider than the screen, the viewport

• This is why you often have to pinch, zoom, and pan to move around a mobile browser

• To ensure that your site is actually being viewed at the width of the device, and not the viewport, use:


	<meta name="viewport" content="width=device-width, initial-scale=1">

<meta> Tag


	<meta name="viewport" content="width=device-width, initial-scale=1">

"width=device-width"

Controls the size of the viewport; sets it to the device's width

"initial-scale=1"

Controls the zoom level

Responsive Rules of Thumb

Responsive Rules of Thumb

• Use pixels for font size on <body> and other top-level elements (containers, wrappers, etc.)

• Use percents for widths, margins, padding of of major layout elements

• Use em's for typography: <h1>, <p>, etc.

• Include the viewport <meta> in the <head>

CSS Frameworks

CSS Frameworks

Framework:

• Standardized approach for solving common problems

• Pre-packaged code used to support the development of website

• Goals:

⇒ Provide a common structure (boilerplate)

⇒ Eliminate repetitive work

⇒ Speed development process

CSS Frameworks

Front End Framework:

• Files and folders of standardized code (HTML/CSS/JS)

• Typography and pre-built styles for HTML elements

• CSS grid for developing a responsive layout

• Solutions to cross-browser compatibility issues

• JS library for UI/UX interactions (dropdowns/carousels/etc.)

CSS Frameworks

Two types of framework:

1) Grid/Layouts - Simple:

⇒ 960 Grid System

⇒ Toast

⇒ Base

2) Complete - Complex:

⇒ Bootstrap

⇒ (Zurb) Foundation

⇒ Skeleton

CSS Frameworks - Selecting a Framework

Rule of thumb:

⇒ The more stylistically complex (the more custom CSS you will be writing), the less complex of a framework you will need

As with plugins:

⇒ Open source

⇒ Recently updated

⇒ Community of active contributors

⇒ Well-documented

CSS Frameworks - Pros/Cons

Pros:

⇒ Major increase in speed of development

⇒ Solutions to common problems already built

⇒ Cross-browser compatibility

Cons:

⇒ Reliance (you will un-learn CSS!)

⇒ Dependence - what if you want to switch frameworks?

⇒ Bloat - unused framework code

⇒ Identical/similar-looking/clones

Bootstrap - The Grid

Core CSS Classes:

<container>

⇒ Top-level element; holds all other code

<row>

⇒ Horiztonal row used to hold col's

⇒ Must be children of container's

<col>

⇒ Used to hold content (text/images/etc.)

⇒ Must be immediate children of row's

Bootstrap - Columns

• Bootstrap's grid is divided into 12 columns

• Columns are created using CSS classes

• Build columns using the following formula:

col-{screen-size}-{width}

<div class="col-md-6">

⇒ A div for medium-sized screens that is 6-columns wide

<div class="col-xs-2">

⇒ A div for x-small screens that is 2-columns wide
